1. Important: what saves where

You change this in WordPress…Stored inUpdated by FTP deploy?
Products, prices, stockDatabaseNo
Banner slides (Customizer)DatabaseNo
Media Library uploadsServer uploads folderNo
Theme code (PHP/CSS)Theme files on serverYes (deploy script)
Rule: The deploy script only uploads theme and plugin files. It does not copy your products, banners, or media. Those live in WordPress on Hostinger.

After you change banners or products, click Publish / Update in wp-admin. If the live site looks old, purge cache (see §10 Troubleshooting).

3. Add a new product

Step-by-step

  1. Log in to wp-admin
  2. Go to Products → Add New
  3. Fill in the fields below
  4. Click Publish

  • Handmade Collection, Instagram Collection, Latest Collection

Make product visible on shop — checklist

  • Status = Published
  • Product image is set (not placeholder)
  • In stock (quantity > 0)
  • At least one category selected
  • Price is set

Edit an existing product

Products → All Products → click product name → change fields → Update

4. Update homepage banner slides

  1. Appearance → Customize
  2. Open Homepage Hero
  3. For each slide (Hero slide 1 … 5): Select or upload image — use 1920 × 720 px
  4. Optional: set Hero “from” price (₹) — leave 0 for auto lowest price
  5. Click Publish (required!)
  6. Hard refresh homepage (Ctrl+F5) or use incognito window
  7. If cached: LiteSpeed Cache → Purge All

Note: Theme deploys no longer reset banner slides (v4.6.50+). Your Customizer choices stay after code updates.

5. Product of the Day

The Product of the Day block highlights one product on the homepage.

How the site picks it

  1. First: Latest Featured product that is in stock with an image
  2. If none: Newest in-stock product with an image

Control it with the Featured flag — no separate SKU picker.

Set today’s product

Option A: Products → All Products → open product → check Featured → Update

Option B: Click the star icon on the product row in All Products

To rotate: uncheck Featured on old product, enable on new one.

Show or hide section

Appearance → Customize → Homepage Sections → toggle Product of the Day → Publish

6. Verify UPI orders (admin)

Customers pay via UPI after placing the order. New orders start as Pending payment.

Morning routine

  1. WooCommerce → Orders
  2. Filter: Pending payment
  3. Process oldest orders first

Per-order verification

StepAction
1Open order — note Order # and total
2Check if customer clicked “I've completed UPI payment” (order note / UTR)
3Open PhonePe / GPay / Paytm business app
4Find payment by order # in remarks, amount, date, or UTR
5Confirm amount exactly matches order total
6If valid → status Processing → Update
7If invalid → Order note → WhatsApp customer → Cancelled if unresolved

Common payment issues

IssueWhat to do
Wrong amount paidWhatsApp for balance or partial refund
No order # in UPI remarksMatch by amount + time + name
Duplicate UTRInvestigate before Processing
Customer never paidLeave Pending or Cancel after follow-up

7. Update order status & shipping

StatusMeaningWhen to use
Pending paymentOrder placed; UPI not verifiedDefault after checkout
ProcessingPayment confirmed; packingAfter UPI verified
CompletedDelivered / closedAfter courier delivery
CancelledOrder voidedInvalid payment / request
On holdAwaiting infoOptional

Ship an order

  1. Open order in Processing
  2. Pack — print slip: order → Print
  3. Add Order note (tick “Note to customer”) with tracking, e.g.:
Shipped via Delhivery. Tracking: 1234567890. Expected delivery 5–7 business days.
  1. Status Completed → Update

Bulk: Select orders → Bulk actions → Change status to processing

On hold / PendingAfter order — UPI pay instructions

Admin — find an order

WooCommerce → Orders — search by #, name, email, or phone; filter by status/date.

Delivery timeline (customer-facing)

  1. UPI verified — within 24 hours
  2. Packed & shipped — tracking in email when available
  3. Delivered — typically 5–10 business days across India

9. Other common tasks

  • Store contact: Appearance → Customize → Store UI → Publish
  • Promo codes: WooCommerce → Marketing → Coupons → Add coupon
  • Hide product: Stock 0 / Out of stock, or set to Draft
  • Best Sellers: Mark products Featured
  • Purge cache: LiteSpeed Cache → Purge All (after banner changes)
  • UPI settings: WooCommerce → Settings → Payments → Manual UPI → Manage
  • Email (SMTP): Install WP Mail SMTP if emails don’t arrive

11. Daily checklist

Morning (15–30 min)

  • WooCommerce → Orders → process Pending payment
  • Verify UPI → move to Processing
  • Reply to WhatsApp / contact messages

When adding new jewellery

  • Upload square image 800×800+
  • Set price, stock, category
  • Publish; check shop + mobile
  • Optional: mark Featured for Product of the Day

When changing homepage

  • Customizer → Homepage Hero
  • Publish Customizer
  • Purge cache; verify homepage

Weekly

  • Review low-stock products
  • Rotate Featured product for Product of the Day
  • WooCommerce → Analytics → Orders
